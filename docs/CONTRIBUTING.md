# Contributing

Quick rules
- Fork -> feature branch -> PR
- Branch names: feat/<desc>, fix/<desc>, chore/<desc>
- Commit messages: imperative, short title, optional body

PR checklist
- Add tests for new behavior (backend: Jest)
- Run lint and tests locally
- Update docs if behavior changes
- Keep PRs focused and small

Code style
- TypeScript in both frontend and backend
- Follow existing conventions in codebase
